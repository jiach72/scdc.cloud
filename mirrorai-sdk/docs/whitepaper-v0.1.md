# 🪞 明镜 Blackbox — 技术白皮书 v0.1

> 版本：0.1.0 | 日期：2026-03-24 | 状态：初稿

---

## 1. 问题陈述

AI Agent 正在从实验室走向生产环境，但当前面临一个核心信任问题：

> **Agent 做了什么？为什么这么做？怎么证明它没被篡改？**

现有方案的不足：
- **日志系统**（ELK/Loki）→ 只记录原始输入输出，不含推理过程
- **APM工具**（Datadog/Sentry）→ 关注性能指标，不关注决策链
- **LLM观测平台**（LangSmith/Langfuse）→ 面向开发者调试，不提供法律级证据

## 2. 解决方案

**明镜 Blackbox** — 专为 AI Agent 设计的行为证据平台。

核心理念：**每只龙虾都该有一个黑匣子**

就像飞机的黑匣子（Flight Data Recorder），Blackbox 在 Agent 运行时默默记录每一次决策，确保数据真实、完整、不可篡改。

### 2.1 四层架构

```
┌─────────────────────────────────────────────────┐
│                用户的 Agent 代码                  │
│         (OpenAI / LangChain / 自定义)             │
└───────────────┬─────────────────────────────────┘
                │ npm install @lobster-academy/blackbox
                ▼
┌─────────────────────────────────────────────────┐
│                 Blackbox SDK                    │
│                                                 │
│  Recorder ──→ Redactor ──→ Signer ──→ Reporter │
│  (录制器)      (脱敏器)    (签名器)    (报告器)   │
└───────────────┬─────────────────────────────────┘
                │ 可选上传
                ▼
┌─────────────────────────────────────────────────┐
│             Lobster Cloud (Phase 2)             │
│  API Gateway + PostgreSQL + Redis + 评测引擎    │
└─────────────────────────────────────────────────┘
```

### 2.2 核心模块

#### Recorder（录制引擎）
- 记录 Agent 的输入、推理过程、输出、工具调用
- 支持手动录制和自动拦截（OpenAI/LangChain 插件）
- 每条记录生成 SHA256 哈希

#### Redactor（脱敏引擎）
- 内置 7 种 PII 模式：email、phone、creditCard、ssn、idCard、ip、url
- 支持自定义正则
- 深度脱敏：递归处理对象/数组
- API密钥等敏感字段名自动拦截

#### Signer（签名引擎）
- Ed25519 数字签名（基于 tweetnacl，零依赖）
- 用户本地持有私钥，签名在本地完成
- 公钥可分发给验证方

#### Reporter（报告引擎）
- 统计摘要：决策数、工具调用、错误率、平均延迟
- 异常检测：高延迟、错误激增、PII泄露
- 支持 JSON/文本/PDF 格式

## 3. 安全设计

### 3.1 数据处理

| 数据类型 | 处理方式 |
|----------|----------|
| 用户输入 | 脱敏后存储（PII → [REDACTED]） |
| Agent 推理 | 完整存储（核心价值） |
| 工具参数 | 脱敏敏感字段 |
| 工具结果 | 摘要存储 |
| API 密钥 | **永不录制** |
| 签名私钥 | **用户本地持有，不上传** |

### 3.2 签名验证链

```
记录哈希 = SHA256(id + agentId + timestamp + input + output)
记录签名 = Ed25519.sign(记录哈希, 用户私钥)

报告签名 = Ed25519.sign(SHA256(report摘要), 用户私钥)

验证方使用用户的公钥验证签名 → 证明记录未被篡改
```
