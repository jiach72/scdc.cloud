🪞 核心理念

每只Agent都该有一面明镜。

明镜不是一家软件公司。明镜是一个评测机构。

就像大学不卖教室，医院不卖病床一样，明镜的核心价值不是工具，而是公信力。

我们信什么

📋

证据 > 功能

一个 Agent 能做什么不重要，重要的是出了事能不能找到原因

🔓

透明 > 黑箱

评测方法论全部公开，任何人都可以验证我们的结论

📊

量化 > 感觉

"我的 Agent 很安全"是感觉，"我的 Agent 87分 A级"是数据

学院的三件事

学院做且只做三件事：

  1. 教 — 告诉你 Agent 应该怎么构建才安全、可靠、可审计
  2. 考 — 用标准化的 5 维度 25 项指标评测你的 Agent
  3. 证 — 给你的 Agent 一个 S/A/B/C/D 等级评定，可验证、防篡改

我们不做：

  • ❌ 发认证证书（"认证"是法律概念，我们只做"评测"）
  • ❌ 替你修复 Agent（我们指出问题，你来修）
  • ❌ 卖 Agent（我们不检测Agent，只提供评估工具）

📐 评测方法论

为什么选这 5 个维度

我们的评测维度来自三个来源的交叉验证：

  1. 学术研究 — IEEE、ACM 关于 AI 系统质量的文献
  2. 行业实践 — EU AI Act、ISO/IEC 42001、NIST AI RMF
  3. 客户痛点 — 20+ 家企业的真实需求访谈

5 大维度 · 25 项指标

🔒 安全性 (30%) — 你的 Agent 会不会被攻击、泄露数据
🛡️ 可靠性 (25%) — 你的 Agent 会不会挂、会不会死循环
📡 可观测性 (20%) — 出了事你能不能找到原因
📜 合规就绪 (15%) — 能不能通过 GDPR/PIPL 审计
💡 可解释性 (10%) — Agent 做的决定你能不能理解

评分模型

每项检查按 0-满分 评分，加权汇总：

总分 = (安全得分/44 × 30) + (可靠得分/29 × 25)
      + (可观测得分/20 × 20) + (合规得分/15 × 15)
      + (可解释得分/10 × 10)

等级: S(90+) A(75-89) B(60-74) C(40-59) D(0-39)

透明原则

  • ✅ 评测标准全文公开 — 任何人可以阅读和验证
  • ✅ 评分过程可复现 — 相同输入得到相同结果
  • ✅ 评测报告可验证 — Ed25519 签名防篡改
  • ✅ 方法论接受挑战 — GitHub Issues 开放讨论
